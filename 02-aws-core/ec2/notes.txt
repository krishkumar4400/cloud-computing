AWS IAM (AWS Identity & Access Management):

    using this service we can manage users on our AWS acoount because let's say in an organization we are working in a group or a team so multiple users should have access so management, creation, and permission of those users are managed by IAM. 

Simple: IAM controls who can access what in AWS

Real life example: 
    AWS account = office
    IAM = security system
    Users = employees
    Permissions = access rights

Example:
    Dev → access EC2
    Admin → full access
    Intern → limited access

IAM is a service that helps you securely control access to AWS resources.

It allows you to manage users, roles, and permissions to define who can access what within your AWS environment.


Core Components of IAM

1. Users: Individual person or service

Example:
Krish (you)
Developer

Each user has:

- Username
- Password / Access keys

2. Groups: Collection of users

Example:
- Dev Team
- Admin Team


Easy management:

Ek baar group ko permission do → sab users ko mil jaata hai

3. Policies (MOST IMPORTANT): JSON rules that define permissions

Simple:

“Policy = what you can do”

Example:
- Allow EC2 access
- Deny S3 delete

4. Roles: Temporary permissions

Example:
- EC2 instance → S3 access
- No login required


How IAM Works (Flow):
    User → IAM → Check Policy → Allow / Deny → AWS Service

Example:
- You try to launch EC2
- IAM checks permission
- If allowed → success
- If not → denied

Types of Access in IAM
1. Console Access: Login via browser
2. Programmatic Access: Access using API / CLI

Example:
    Suppose:
- User = Krish
- Policy = Only EC2 allowed

Result:
    Can launch EC2 ✅
    Cannot access S3 ❌

Best Practices:
    - Don’t use root account
    - Use roles instead of access keys
    - Follow least privilege
    - Use MFA (Multi-Factor Authentication)


“IAM ensures that the right user has the right access to the right resource at the right time.”

- Free Service: IAM is offered at no additional cost 
- Global service 
- Root account created by default, shouldn't be used or shared


- What we can do in IAM service: 
    - Create users: You can create individual user accounts for people who need access to your aws resources. 
    
    - Assign Permissions: You can assign specific permissions to users, groups, or roles to control what actions they can perform on aws services.

    - Create Groups: You can group users together and assign permissions to the group, making management easier for multiple users.
